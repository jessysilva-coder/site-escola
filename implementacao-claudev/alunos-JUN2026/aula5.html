Primeiros conceitos

O que é HTML e CSS?

HTML

HTML é a estrutura da página. Ele organiza títulos, textos, imagens, botões, cards, tabelas, links e seções.

CSS

CSS é a aparência da página. Ele define cores, tamanhos, espaçamentos, fontes, bordas, sombras, responsividade e organização visual.

💡

Pense assim: o HTML é o esqueleto da página e o CSS é o visual que deixa tudo bonito, organizado e profissional.

Publicação

O que é GitHub Pages?

🌐

GitHub Pages é um recurso gratuito do GitHub para publicar páginas na internet. Ele transforma arquivos HTML, CSS e JavaScript em um site acessível por link, sem contratar hospedagem para projetos simples.

Como criar o repositório e configurar o Pages

1

Acesse o GitHub

Entre em sua conta e clique em New repository.

2

Nomeie o repositório

Use o nome gestao-de-atendimentos. Prefira nomes sem acento, sem espaço e em letras minúsculas.

3

Defina como público

Marque o repositório como Public para facilitar a publicação.

4

Crie o arquivo principal

Adicione o arquivo index.html. Esse será o arquivo inicial da página.

5

Adicione os arquivos do projeto

Envie imagens e outros arquivos. Adicione ao repositório todos os arquivos utilizados na página.

6

Ative o GitHub Pages

Vá em Settings → Pages, selecione a branch main e a pasta /root. Depois clique em Save.

7

Acesse o link publicado

Após alguns minutos, o GitHub exibirá o link da página publicada.

Estrutura de publicação

Domínio, hospedagem e registro

🌍

Para colocar uma página na internet, precisamos entender três elementos que trabalham juntos. Primeiro registramos o nome, depois escolhemos onde os arquivos ficarão hospedados e, por fim, conectamos o domínio ao provedor.

1. Registro do domínio

É a compra e reserva do endereço que será usado no site, como minhaempresa.com.br. No Brasil, um dos serviços mais conhecidos é o Registro.br.

2. Provedor ou hospedagem

É o serviço que mantém os arquivos da página disponíveis na internet. O GitHub Pages funciona como uma hospedagem gratuita para projetos em HTML, CSS e JavaScript.

3. Conexão do domínio

Depois de registrar o domínio e publicar a página, configuramos o endereço para apontar para a hospedagem. Assim, o visitante usa o domínio personalizado para acessar o conteúdo.

Resumo rápido

O domínio é o endereço, a hospedagem é o local onde os arquivos ficam e o registro é o processo que garante que esse endereço pertence a você.

Exercício 2

Criação de sistemas: passo a passo

🛠️

Um sistema simples pode ser construído em etapas. Primeiro definimos a lógica e os dados. Depois criamos a conexão, a interface e a publicação.

A

Claude para estruturar o sistema

Transforma a ideia em escopo: objetivo, usuários, funcionalidades, regras, telas e fluxo de uso.

B

Google Sheets como banco de dados

A planilha armazena os registros do sistema. Cada aba funciona como uma tabela de dados.

C

Apps Script para conexão da API

Cria a ponte entre a página web e o Google Sheets para cadastrar, consultar, editar e excluir registros.

D

Claude para gerar a aparência da página

Cria HTML, CSS e JavaScript para montar a interface do sistema.

E

GitHub para publicar

Publica a interface do sistema pelo GitHub Pages, permitindo acesso por link.

Depois do prompt

Próximos passos com a resposta do Claude

🧭

Depois de enviar o prompt do sistema para o Claude, ele vai te orientar por etapas. A ideia é não tentar fazer tudo de uma vez: primeiro você cria a base, depois conecta com o Apps Script, testa o sistema e só então publica no GitHub Pages.

1

Crie o arquivo do Google Sheets

Crie o arquivo do Google Sheets conforme a orientação do Claude, usando a estrutura sugerida: nome do arquivo, abas, colunas e exemplos de preenchimento.

2

Cole o código no Apps Script

Abra o Apps Script a partir da planilha, cole o código gerado pelo Claude e clique em Executar para iniciar os testes e autorizações necessárias.

3

Implante como API de registro

Clique em Implantar para transformar o Apps Script em uma API de registro. Essa etapa cria o caminho de comunicação entre a página web e a planilha.

4

Crie uma nova implantação como aplicativo da web

Crie uma nova implantação na opção de app da web e preencha as configurações conforme a imagem. Essa configuração é o que permite que a página envie e receba dados.

5

Envie a URL para o Claude

Depois da implantação, copie o link da URL do aplicativo da web e envie para o Claude. Com esse link, ele poderá gerar o sistema já conectado ao Google Sheets.

6

Teste o sistema gerado

Abra o sistema, cadastre registros de teste, edite informações, aplique filtros e confira se os dados estão sendo enviados corretamente para o Google Sheets.

7

Publique no GitHub

Após os testes e validações, publique o sistema no GitHub Pages para que ele possa ser acessado por link.

Importante

Teste com dados fictícios antes de usar informações reais. Assim você valida o fluxo, evita erros e protege dados sensíveis durante a construção do sistema.
